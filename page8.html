Attacks By Philip Wu

When something has vulnerabilities, it can be easily exploited by hacking groups. Microsoft is a common target for this, thanks to the widespread use of it's software. For example, one exploit allowed remote code to be executed on a computer through Internet Explorer. [CVE-2018-8174]

There are many different types of attacks that can be performed, which is usually done by the hacker, though rare instances may include the attack being performed by a script set up by the hacker on a website.

One famous example of an attack is SQL injection.

SQL injection is where a vulnerability allows attackers to interfere with queries that an application makes to a database. Generally speaking, it allows attackers to access and view data that they are not normally able to access. This can include data belonging to other users, or any other data that the application is able to access. In many cases, an attacker can even modify or delete this data, resulting in persistent changes to the behavior of an application.

These attacks have many uses, such as:
Compromising the server a database is hosted on, or similar back-end infrastructures. 
DoS (Denial of Service) attacks.

Successful SQL injection attempts can result in unauthorized access to sensitive data, such as passwords, credit card details, or similar personal user information. Many high-profile data breaches in recent years have been a result of SQL injection attacks, leading to reputational damage and fines. 

In some severe cases, attacks can gain persistent backdoor access to an organization's systems, leading to long-term compromisation that can easily go unnoticed.

SQL injection examples:
1. Retrieving hidden data, where a SQL query is modified to return additional results.
2. Subverting application logic, where a query is changed to interfere with an application's logic./
3. UNION attacks, where data can be retrieved from different locations in a database.
4. Examining the database, where information about the database such as the version and structure is extracted for easier access and further exploitation.
5. Blind SQL injection, where the results of a query that you control are not returned to you by the application's response system.

Another example of attacks over the Internet are ping attacks. These involve attacking a system through overloading it with information. 

For example, a ping of death attack attacks a system by pinging it with fragmented packets of an IP size of over the maximum of 65,535 bytes. When the receiving system reassembles the packet, it can experience buffer overflows and other crashes.

Another instance of these attacks is Botnets.
Through the use of millions of systems infected by malware under a hacker's control, a hacker can carry out Distributed Denial of Service attacks. These systems are used to carry out attacks against systems, overwhelming their bandwidth and processing capabilities. These attacks are difficult to trace thanks to systems of botnets being located in differing geographic areas throughout the world.

Takeaways: 
- Check if there are any patches you can apply to your software, especially if it's a popular software
- When creating your own databases, ensure that input is cleaned to make sure that it cannot interfere with the structure of your code.
- Suspicious links often leak your IP, which can lead to attacks being targeted towards you.
